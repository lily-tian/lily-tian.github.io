XOR Encryption


Exclusive or (XOR) is a logical operation that return true if inputs are different and false if inputs are the same. In bitwise operations, or operations at the individual bit (0 or 1) level, two patterns of equal length are taken and XOR applied on each pair of corresponding bits.

For example, take the letter "A". This is an ASCII character that has a corresponding binary 01000001. Then take the digit "8", or 00111000. The bitwise XOR will first compare the first bit of both patterns, 0 and 0. As they are the same, it will return 0. The next bits in both sequences, 1 and 0, are different and thus the XOR returns 1. Continuing this, it can be seen that "A" XOR "8" is 01111001, which corresponds to the lower case letter "y".

A:  01000001
8:  00111000
---------------
y:  01111001

One useful feature of XOR is that applying the same operation again will retrieve the initial value. In the example above, the XOR of "y" and "8" is 01000001, or the original letter "A". This effectively allows for simple encryption and decryption of data with a single key.

y:  01111001
8:  00111000
---------------
A:  01000001

However, in cryptography, XOR alone is insufficient. The key - "8" in this case - must not remain fixed or it will be easy to uncover the connection between the plaintext and ciphertext. One way to hack the encryption is through brute force, or simply trying every possible key until a coherent message appears. There are also alternative methods, such as frequency analysis. For example, assume the language of the text is English. So it is with high probability that the letter "e" will appear with greater frequency than all other letters. Finding the XOR of "e" and the encrypted character of highest frequency can reveal the key, which can then be used to decrypt the rest of the text.

Using a changing key is one way to overcome this problem, which leads us to the linear-feedback shift register.

The Linear-Feedback Shift Register


A linear feedback shift register stores a sequence of bits then performs a series of discrete step operations on those bits. As illustrated in the diagram below, the LFSR first requires an initial seed, or any sequence of bits. It then finds the XOR of the leftmost bit and the bit at the "tap" position, places the result to the right, removes the leftmost bit, and shifts all the bits to the left by one.

Image

Repeating these steps will generate a new sequence of bits every time, which can then be put together to create pseudo-random keys used for encryption. As long as you know the initial seed and the tap location, you can use the LFSR to produce the same keys that will decrypt the ciphertext.
